# Skill: Build Release Guard

## 触发条件

当任务涉及 Release Guard Lite 的功能开发、bug 修复、或模块实现时使用此剧本。
不相关的任务（如文档讨论、产品策略讨论）不要使用。

## 执行流程

### Step 1: 复述任务

用一句话复述当前要做什么，确认理解一致。

### Step 2: 确认范围

回答以下问题：
- 这个任务在 MVP 5 件事之内吗？
- 需要修改哪些文件？
- 需要新增文件吗？如果是，理由是什么？
- 需要新增依赖吗？如果是，理由是什么？

如果任务超出 MVP 范围，明确指出并请求确认。

### Step 3: 识别最小改动集

- 列出需要改动的文件（不超过必要范围）
- 对每个文件说明改什么
- 不做任何"顺手"的改进

### Step 4: 实施

- 按最小改动集执行
- 每个文件改完后立即验证
- 遇到意外情况立即停下来说明

### Step 5: 验证

- 给出验证命令
- 如果有测试，运行测试并报告结果
- 如果是 P0 模块（配置解析、风险分级、报告生成），必须有测试覆盖

### Step 6: 输出报告

按 `.claude/rules/output.md` 的要求输出：
1. 改动文件清单
2. 每个文件改了什么
3. 为什么改
4. 可能影响什么
5. 如何验证

## 禁止事项

- 不做范围外的"顺手优化"
- 不引入未经批准的依赖
- 不跳过验证步骤
- 不在没有测试的情况下宣称 P0 模块完成
- 不把 AI 判断放在确定性规则前面
