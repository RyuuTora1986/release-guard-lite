How It Works

1

Read Git Diff

Compares your feature branch against the base branch to find all changed files

2

Run Checks

Executes your configured commands — lint, test, typecheck, or anything else

3

Classify Risk

Deterministic rules, not AI guessing. High-risk file + failed check = HIGH risk

4

Report + Alert

Markdown report saved locally. Discord embed notification sent to your team

Frequently Asked Questions

What is Release Guard Lite?

A free, open-source command-line tool that checks your code changes for deployment risk before you release. It reads your git diff, runs your checks, classifies risk as LOW/MEDIUM/HIGH, and generates a report.

Does it use AI to judge risk?

No. Risk classification uses deterministic rules: which files changed + whether checks passed. No LLM guessing, no hallucination risk. The rules are transparent and configurable.

How do I install it?

Run npx release-guard-lite init to generate a config file, then npx release-guard-lite run to check your changes. Or install globally with npm install -g release-guard-lite.

Can I use it in GitHub Actions or other CI/CD?

Yes. Exit code 0 = low risk, 1 = medium/high. Add one line to your workflow. Use --json for machine-readable output that integrates with any CI pipeline.

What if I don't use Discord?

Discord is optional. Leave the webhook blank — you still get the local Markdown report. Use --json to pipe results to Slack, email, or any notification system you prefer.

What files are considered high risk?

By default: package.json, lock files, .env files, migration directories, authentication modules, payment code, and deployment configs. All patterns are configurable in release-guard.config.json.

Is it free?

Yes, fully open-source under the MIT license. Free for personal and commercial use. No signup, no account, no tracking.

How is this different from just running tests?

Tests tell you if code works. Release Guard tells you if the change is risky — combining file-level risk patterns, check results, and a checklist into one report. It's the difference between "tests pass" and "this is safe to ship."
